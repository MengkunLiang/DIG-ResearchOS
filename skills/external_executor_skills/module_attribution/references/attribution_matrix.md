# Module Attribution Matrix

Attribute outcomes to modules, not only to whole methods.

For ours:

- module id
- planned mechanism
- code path
- ablation or diagnostic run
- positive, neutral, negative, or inconclusive effect
- supported mechanisms
- unsupported mechanisms

For baselines:

- encoder or representation module
- adaptation or training loss
- regularizer
- treatment or task representation
- evaluation protocol
- why this module may explain baseline strength

The output should support idea refinement: keep, modify, drop, or narrow boundary.
