# Figure And Table Contract

There are two figure classes.

Method framework figure:

- comes from realized method package, code structure, module attribution, and claim boundary
- shows only implemented modules
- maps figure nodes to code paths
- maps emphasized modules to ablation or diagnostic evidence when available
- must not show dropped, unimplemented, or unsupported modules as core contributions

Result figures and tables:

- come from raw results
- map every number to source result, config, log, run id, and seed
- include plot script or generation notes
- state evidence level

Each figure/table inventory item needs:

- id
- type
- claim supported
- source result
- source config
- source log
- plot script or editable source
- caption draft
- evidence level
