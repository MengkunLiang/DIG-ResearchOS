# T3.5 文献综合实现文档

## 概述

Reader Agent（综合模式）是ResearchOS pipeline中T3和T4之间的桥梁，负责综合所有论文笔记产出系统性的文献综述。它读取T3产出的所有paper_notes，结合comparison_table和missing_areas，产出包含5个必需章节的synthesis.md，为后续的idea生成（T4）提供高质量的文献分析。

**在Pipeline中的位置**: T3.5（文献综合阶段）

**代码位置**: 
- Agent实现: `/home/liangmengkun/ResearchOS/researchos/agents/reader.py`（synthesize模式）
- Prompt模板: `/home/liangmengkun/ResearchOS/researchos/prompts/reader.j2`（synthesize模式部分）

## 输入

### 必需输入
- `project.yaml`: 项目配置文件（包含研究方向）
- `literature/paper_notes/`: T3产出的所有论文笔记（至少10篇）
- `literature/comparison_table.csv`: 论文对比表

### 可选输入
- `literature/missing_areas.md`: T2阶段标注的文献缺口

### 输入格式示例

**paper_notes/** 目录结构:
```
literature/paper_notes/
├── arxiv_2301.12345.md
├── arxiv_2302.67890.md
├── s2_abc123.md
└── ...
```

**comparison_table.csv**:
```csv
id,title,year,venue,method_family,dataset,key_metric,metric_value,baseline_of_ours,relevance_score
arxiv_2301.12345,Efficient Transformers,2023,ICML,Attention Optimization,ImageNet,Top-1 Acc,85.3,Yes,9
```

## 输出

### 产出文件

**literature/synthesis.md**: 包含5个必需章节的综合文献综述

## Workflow

### 执行流程

#### Step 1: 读取所有输入
- 使用 `list_files` 列出 `literature/paper_notes/` 目录下的所有笔记
- 使用 `read_file` 读取所有笔记内容
- 读取 `comparison_table.csv` 和 `missing_areas.md`（如果存在）

#### Step 2: 分析方法家族
将所有论文按方法聚类成3-5个家族：
1. 识别共同的技术特征（如"基于注意力优化"、"知识蒸馏"、"模型压缩"等）
2. 为每个家族选择2-4篇代表性论文
3. 总结每个家族的核心假设/技术思想
4. 列出每个家族的典型benchmark和结果

#### Step 3: 识别共同假设
列出领域内普遍默认但**可能值得质疑**的2-4个假设：
1. 从paper_notes中提取各论文的隐含假设
2. 识别哪些假设被多篇论文共同依赖
3. 分析每个假设的潜在问题
4. 评估挑战这些假设的创新价值

**这部分通常藏着innovation机会！**

#### Step 4: 分析性能-效率前沿
从comparison_table中提取性能和效率数据：
1. 识别在性能-效率权衡上表现最好的论文（Pareto前沿）
2. 分析不同方法的权衡策略
3. 识别是否存在"高性能+高效率"的空白区域

#### Step 5: 识别技术趋势
从时间维度分析2-3个明显趋势：
1. 对比近12个月 vs 更早期的论文
2. 识别技术路线、评估方法、新任务等方面的变化
3. 预测未来可能的发展方向

#### Step 6: 提出可操作研究问题
列出3-6个可推进的研究问题：
1. 基于方法家族、共同假设、前沿分析、趋势识别
2. 结合missing_areas中的缺口
3. 每个问题必须包含：Why it matters, Why unsolved, Potential angle of attack, Related papers

#### Step 7: 写入synthesis.md
使用 `write_file` 写入完整的synthesis.md，并调用 `finish_task` 完成任务。

## 关键实现细节

### 1. 五个必需章节详解

#### 章节1: 方法家族分类
**目标**: 将所有论文按方法聚类成3-5个家族

**要求**:
- 每个家族包含2-4篇代表性论文
- 明确每个家族的核心假设/技术思想
- 列出典型benchmark和结果
- 使用规范化的论文ID引用（如 `[arxiv_2301.12345]`）

**示例**:
```markdown
### 家族1: 基于注意力优化的方法
- **代表论文**: [arxiv_2301.12345], [s2_abc123]
- **核心假设**: 注意力机制是Transformer的性能瓶颈
- **技术思想**: 通过稀疏注意力、线性注意力等方法降低复杂度
- **典型benchmark**: ImageNet Top-1 Acc 85.3%, 推理时间45ms
```

#### 章节2: 共同假设
**目标**: 列出领域内普遍默认但可能值得质疑的2-4个假设

**要求**:
- 每个假设包含：假设描述、支持证据、潜在问题、挑战价值
- 重点关注**可能不成立或有局限**的假设
- 这部分通常藏着innovation机会

**示例**:
```markdown
### 假设1: 更大的模型总是更好
- **支持证据**: [arxiv_2301.12345], [arxiv_2302.67890] 都使用大规模模型
- **潜在问题**: 忽略了效率和部署成本，小模型+好方法可能更实用
- **挑战价值**: 如果能用小模型达到大模型性能，将大幅降低部署成本
```

#### 章节3: 性能-效率前沿
**目标**: 分析论文在性能和效率两个维度上的分布

**要求**:
- 识别Pareto前沿论文
- 分析不同方法的权衡策略
- 识别空白区域（高性能+高效率）

**示例**:
```markdown
### 前沿论文
- [arxiv_2301.12345]: Top-1 Acc 90.5%, 推理时间200ms（高性能但低效率）
- [arxiv_2302.67890]: Top-1 Acc 85.3%, 推理时间45ms（高效率但中等性能）
- [s2_abc123]: Top-1 Acc 88.0%, 推理时间80ms（平衡点）

### 空白区域
目前没有论文同时达到 >90% Top-1 Acc 和 <50ms 推理时间，这是一个潜在的创新空间。
```

#### 章节4: 技术趋势
**目标**: 从时间维度分析2-3个明显趋势

**要求**:
- 每个趋势包含：趋势描述、时间线、代表论文、未来方向
- 对比近12个月 vs 更早期的论文
- 预测未来可能的发展方向

**示例**:
```markdown
### 趋势1: 从全局注意力到稀疏注意力
- **时间线**: 2023年前主要使用全局注意力，2023-2024年稀疏注意力成为主流
- **代表论文**: [arxiv_2301.12345]（2023早期，全局注意力）, [arxiv_2312.67890]（2023晚期，稀疏注意力）
- **未来方向**: 可能进一步发展为自适应稀疏注意力
```

#### 章节5: 可操作研究问题
**目标**: 列出3-6个可推进的研究问题

**要求**:
- 每个问题包含：Why it matters, Why unsolved, Potential angle of attack, Related papers
- 问题必须是**可操作的**（有明确的研究路径）
- 基于前面4个章节的分析

**示例**:
```markdown
### Q1: 如何在保持高性能的同时大幅降低推理时间？
- **Why it matters**: 当前高性能模型推理时间过长，限制了实际部署
- **Why unsolved**: [arxiv_2301.12345] 等论文只关注性能，忽略了效率；[arxiv_2302.67890] 等论文虽然高效但牺牲了性能
- **Potential angle of attack**: 结合稀疏注意力和知识蒸馏，在小模型上达到大模型性能
- **Related papers**: [arxiv_2301.12345], [arxiv_2302.67890], [s2_abc123]
```

### 2. 引用规范

**必须使用规范化的论文ID**:
- 使用paper_notes中的文件名（特殊字符替换为下划线）
- 例如：`[arxiv_2301.12345]` 而不是 `[arxiv:2301.12345]`
- 例如：`[s2_abc123]` 而不是 `[s2:abc123]`

**禁止编造引用**:
- 所有引用的论文必须存在于paper_notes中
- 不要引用不存在的论文ID

### 3. 长度控制

- **最短**: 2000字符（过短说明没有认真综合）
- **推荐**: 3000-8000字符
- **最长**: 无硬性限制，但避免写流水账

### 4. 批判性思维

**不要只是罗列论文**，要：
- 分析：识别模式、共同点、差异
- 对比：比较不同方法的优劣
- 找出空白：识别未被解决的问题
- 提出问题：为T4 Ideation准备创新机会

### 5. 面向T4

记住这份综述是为T4 Ideation准备的，要：
- 突出创新机会（共同假设、空白区域、未解决问题）
- 提供可操作的研究问题
- 明确指出哪些方向值得探索

## 常见问题

### Q1: 如何处理笔记数量太少（<10篇）的情况？
**A**: 
1. 检查T3是否正常完成
2. 如果笔记确实太少，可以降低质量要求但仍需包含5个章节
3. 在synthesis.md中标注"笔记数量有限，综述可能不全面"

### Q2: 如何识别方法家族？
**A**: 
1. 从paper_notes的 "Method Overview" 部分提取方法描述
2. 识别共同的技术关键词（如"attention"、"distillation"、"pruning"）
3. 使用comparison_table的 "method_family" 字段作为参考
4. 聚类成3-5个家族（太少说明分类太粗，太多说明分类太细）

### Q3: 如何识别共同假设？
**A**: 
1. 从paper_notes的 "Limitations" 和 "Weaknesses" 部分寻找线索
2. 识别多篇论文都依赖的前提条件
3. 重点关注**可能不成立或有局限**的假设
4. 这些假设通常是创新的切入点

### Q4: 如何分析性能-效率前沿？
**A**: 
1. 从comparison_table提取性能指标（如accuracy）和效率指标（如inference_time）
2. 绘制二维散点图（性能 vs 效率）
3. 识别Pareto前沿（没有其他论文在两个维度上都更好）
4. 识别空白区域（高性能+高效率）

### Q5: 如何提出可操作研究问题？
**A**: 
1. 基于前面4个章节的分析
2. 结合missing_areas中的缺口
3. 每个问题必须有明确的研究路径（Potential angle of attack）
4. 避免过于宽泛的问题（如"如何提升模型性能"）

## 性能指标

- **平均运行时间**: 10-30分钟（取决于笔记数量）
- **Token消耗**: 80K-200K tokens
- **成功率**: >95%（在笔记质量良好的情况下）
- **输出质量**: 包含5个章节，3000-8000字符，至少5个论文引用

## 已知限制

1. **笔记依赖**: 依赖T3产出的笔记质量，如果笔记质量差会影响综述质量
2. **聚类主观性**: 方法家族分类有一定主观性，不同运行可能产生不同分类
3. **趋势识别**: 需要足够的时间跨度（至少12个月），如果论文都是同一年份可能难以识别趋势
4. **创新机会**: 识别创新机会需要深度理解，可能遗漏某些隐藏的机会
5. **Token消耗**: 读取大量笔记会消耗大量tokens，可能触发预算限制
