#### Bug #2: T1 project.yaml schema验证错误 ✅
- **严重程度**: 严重
- **问题**: YAML解析器将日期字符串转换为date对象，导致JSON Schema验证失败
- **修复**: 在validator.py中添加`_normalize_dates_for_validation()`函数，递归转换date/datetime对象为ISO 8601字符串
- **影响文件**: `researchos/schemas/validator.py` (+48行)

#### Bug #3: load_jsonl无法处理错误JSON数组格式 ✅
- **严重程度**: 严重
- **问题**: T1生成的seed_papers.jsonl是JSON数组`[]`而非JSONL格式，导致scout.j2模板报错
- **修复**: load_jsonl()添加类型检查，跳过空数组和非字典对象
- **影响文件**: `researchos/agents/_common.py` (+15行)

#### Bug #4: T2 Scout Agent token预算不足 ✅
- **严重程度**: 中等
- **问题**: max_tokens_total=120K不足，实际使用124K导致提前停止
- **修复**: 增加到200K以支持更多检索迭代
- **影响文件**: `researchos/agents/scout.py` (1行)

#### Bug #5: T4 Ideation Agent使用不存在的LLM profile ✅
- **严重程度**: 致命
- **问题**: llm_profile="deep_reasoning"在model_routing.yaml中未定义
- **修复**: 改为None使用默认profile
- **影响文件**: `researchos/agents/ideation.py` (1行)
